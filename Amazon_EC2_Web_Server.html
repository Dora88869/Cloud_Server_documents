AWS EC2 Setup Guide

Log in to the Amazon EC2 Management Console

To start the lab, log into the Amazon EC2 management console: AWS EC2 Console

For signup purposes, choosing "Business" or "School" and using a Murdoch email has reportedly led to fewer delays with credit card verification.

Launch an Ubuntu Machine in EC2

  • Launch a new instance (virtual machine).
  • Follow the steps on the AWS site.
  • Instance type: Pick a "free tier eligible" Ubuntu 20.04 Instance.
  • Configure instance details: Leave everything as default.
  • Add storage: Default is an 8GB hard drive; leave it as is.
  • Add tags: No tags needed, proceed to security group setup.

Configure Security Group

  • Security group name: Call it "ssh-and-web".
  • The existing **SSH** permission is required for remote management.
  • Click **Add Rule**, then select **HTTP (web)** to allow web traffic.

Review and Launch

  • Ignore any warning about the server being "open to the world"—we’re building a public webserver!
  • Click **Launch** if the configuration looks good.

Create a Key Pair

  • AWS uses **key files** instead of passwords for authentication.
  • Save this key file securely—it’s essential for managing the virtual machine.
  • Name it **"webserver-key"** for easy identification.

Launch Instance

  • Click **Launch Instance** to start the server.
  • Use **View Instance** to monitor progress.
